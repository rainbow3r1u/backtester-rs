### v2: 日线BB混合模式（首次跑日线）

**时间**: 2026-04-29 00:36

- `daily_gain_pct` 改为10.0%
- 买入信号窗口：信号后24小时内
- 现货止损：用当日日线收盘价（**未来函数**）
- 涨幅过滤：检查当日涨幅（**未来函数**）

**结果**:
| 指标 | 数值 |
|------|------|
| 最佳综合 | +184.6% |
| 最差综合 | -132.6% |
| 中位数 | 0.0% |
| 平均 | -4.2% |

**关键问题**: 用当日日线收盘价做止损和涨幅过滤，在盘中就知道了收盘结果，属于**lookahead bias**。

### v4: 修复止损未来函数（昨日日线止损）

**时间**: 2026-04-29 00:45  
**核心改动**:
- `spot.check_positions` 传入 `ts_1d_prev`（前一日日线）而非当日
- `chk_gain` 改为检查前一日涨幅，跳过当日

**预期**: 修复后收益应下降  
**实际结果**:
| 指标 | 数值 |
|------|------|
| 最佳综合 | **+357.5%** |
| 最差综合 | -126.4% |
| 中位数 | +7.6% |
| 平均 | **+51.5%** |
| 正收益 | **56/100** |
| 最佳参数 | p=20, std=2.5, h=2 |

**反直觉分析**:  
修复后收益反而暴涨，原因是：
- v2/v3用当日下轨止损 → 在上升趋势中当日下轨 > 昨日下轨 → 更容易触发止损 → 过早离场
- v4用昨日下轨止损 → 更宽松 → 趋势中持仓更久 → 抓住完整涨幅

**问题**: 虽然止损修复了，但**开仓时间仍有未来函数**（信号后24小时内任意时间开仓，可能盘中提前开仓）。

## 四、最终结论

### 当前最优真实参数（v7）

```
BB:  period=30, std_mult=2.0, min_hours=2, hl_window=3, hl_min=2, daily_gain_pct=20%
VS:  margin=20, max_daily_tp=2, min_ratio=4.0, sl_pct=2%, tp_pct=50%
```

**预期收益**: +109.5%  
**预期回撤**: 57.0%  
**合约交易**: 262笔，胜率38.9%  
**正收益概率**: 36%

### 策略特征

1. **高波动高回撤**：最佳+109%，最差-105%，中位数0%
2. **两极分化**：要么大赚要么大亏，没有中间态
3. **日止盈2次最优**：严格止盈比宽松止盈更能保护利润
4. **涨幅过滤20%最激进**：5%过滤更稳健但上限低

### 未解决问题

1. **BB信号预计算**：`detect_bb_climb` 在回测前就在完整日线上跑完，虽然开仓时间固定在次日00:00，但严格来说信号检测应在收盘后执行
2. **合约双阴过滤**：虽然改为前一小时，但1h K线数据在缓存中也是完整的
3. **平均收益仍为0附近**：策略整体不稳定，需要更长时间验证

### 建议

- **实盘参数**: `gain=5% + max_dtp=2 + margin=20`（稳健取向）
- **或**: `gain=20% + max_dtp=2 + margin=20`（激进取向，但需承受57%回撤）
- **不建议**: h=2（波动太大），h=6/h=8（信号太少）
