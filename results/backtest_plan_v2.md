# BB_CLIMB 现货 + VOL_SURGE 合约 双轨回溯方案

## 背景与目标

重新校准两套策略的定位：

| 策略 | 交易品种 | 定位 | 目标 |
|------|---------|------|------|
| **BB_CLIMB** | 币安现货 | 长线波段趋势判断 | 捕捉布林爬坡的完整趋势，真金白银买入持有 |
| **VOL_SURGE** | 币安永续合约 | 短线脉冲狙击 | 只做15分钟放量突破，吃脉冲就走 |

**核心设计逻辑：** BB现货持仓天然就是VS合约的方向过滤器。现货涨了说明真有人买，VS在这个方向上加杠杆做脉冲。

## 需要回溯验证的核心问题

1. BB_CLIMB 做现货回测，看最佳参数组合
2. VOL_SURGE 做合约回测（即当前第一名参数），看收益和回撤
3. 模拟「VS只在BB现货持仓时开仓」的混合模式，看胜率提升

## 回测范围

- 币种：524个交易对（排除BTCUSDT, ETHUSDT, SOLUSDT, USDEUSDT及所有稳定币）
- 数据：60天历史K线
- 手续费：现货0.1%，合约0.04%

### 合约风控规则（必须完整实现）

1. 日止盈两次过滤：当日同币种止盈≥2次后，当日不再开仓
2. 阴阳质检前置：开仓前检查现货/合约K线阴阳方向，不一致则不开
3. 双阴过滤：最近3根1h K线中≥2根收阴，VOL_SURGE不开仓
4. 日涨幅过滤：当日涨幅>20%的币种跳过
5. 止损冷却：止损后同币种30分钟内不重复开仓
6. 满仓替换：满仓5/5且VOL_SURGE ratio≥5.0时替换最弱非VS持仓
7. VOL_SURGE不可被替换
8. 联合爆仓：总权益≤0时全部强平

## 第三部分：混合模式（唯一需要验证的模式）

> **核心逻辑：BB现货不开仓，VS合约就不开枪。BB现货清了，VS合约同步平仓。**

### 模式变更（仅改变开仓条件，参数和规则不变）

**变更点只有一个：** VS合约开仓前，先检查BB现货是否持有该币种。

**保持不变的部分：**
- BB_CLIMB 参数和规则保持不变（沿用当前配置）
- VOL_SURGE 参数和规则保持不变（沿用当前第一名配置）
- 所有风控规则不变（日止盈两次、阴阳质检、双阴过滤、止损冷却等）
- 仓位管理不变（MAX_POSITIONS=5，满仓替换等）

### 逻辑

```
BB现货按现有参数和规则开仓
         ↓
VS合约检测到15分钟放量信号
         ↓
检查BB现货是否持有该币种
    → 是 → 按现有VS参数开仓合约（参数、保证金、止盈止损不变）
    → 否 → 不开仓
         ↓
BB现货平仓时，VS合约同步平仓（即使未达止盈止损）
```

### 需要改造的回测逻辑

```
for each symbol:
    # BB现货部分：完全不变
    if BB现货信号触发（现有参数和规则）:
        record_spot_entry(symbol, price, amount)
    
    # VS合约部分：只改开仓条件
    if VS合约信号触发（现有参数和规则）:
        if symbol in spot_positions:  # 新增：检查现货是否有仓
            open_futures_position(symbol, margin, leverage)  # 参数不变
        else:
            skip  # 不开仓
```

### 预期效果
- VS信号量减少40-60%（只在BB持仓期间开枪）
- 胜率预期提升
- 整体回撤预期降低
